## 1. 개요

### 1.1 목적
- 실제 사용자 관점에서의 문제 발견
- 입력 흐름 및 결과 해석 가능성 검증

## 4. 피드백 수집

### 4.1 피드백 양식
- 파일: [alpha_feedback_log.md](alpha_feedback_log.md)
- 항목: 시나리오 ID, 발견 내용, 심각도, 스크린샷

### 4.2 이슈 추적
- 파일: [ui_ux_issue_list.md](ui_ux_issue_list.md)
- 분류: UI 버그, UX 개선, 문구 수정, 법적 리스크

### 4.3 심각도 기준

| 등급 | 설명 | 조치 |
|------|------|------|
| Critical | 서비스 불가, 법적 리스크 | 즉시 수정 |
| High | 주요 기능 오류 | 릴리스 전 수정 |
| Medium | 사용성 저하 | 가능한 수정 |
| Low | 개선 권장 | 백로그 등록 |
