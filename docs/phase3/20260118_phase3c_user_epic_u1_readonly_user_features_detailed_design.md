# Phase 3-C / 사용자 기능 Epic (U-1) 상세 설계
최초작성일자: 2026-01-18
최종수정일자: 2026-01-18

## Epic ID
- **Epic**: U-1
- **명칭**: 읽기 전용 사용자 기능 (Read-only User Features)
- **Phase**: Phase 3-C (후반)

---

## 1. 목적 (Purpose)

U-1의 목적은 사용자에게 **행동을 유발하지 않으면서도 서비스의 신뢰성과 가치를 체감**할 수 있는 정보를 제공하는 것이다.

> 핵심 원칙: *결과를 바꾸지 않고, 이해만 돕는다*

본 Epic은 투자 권유·추천·시그널을 제공하지 않으며, 모든 기능은 **조회(Read-only)**로 제한된다.

---

## 2. 사용자 기능 도입 전제 조건

- C-1(운영 안정성) 사고 제어 가능
- C-2(데이터 품질) 출처·재현성 확보
- C-3(성과 분석) 해석 기준 명문화
- C-4(관리자 통제) 검증 완료

→ **사용자 노출 허용 상태**

---

## 3. 기능 범위 (In / Out)

### In-Scope (허용)
- 포트폴리오 구성 조회
- 성과 조회(기간 선택)
- 기준·출처·계산 방식 설명
- 참고용/정보 제공 UI 문구

### Out-of-Scope (금지)
- 종목 추천/교체 제안
- 리밸런싱 알림
- 매매·행동 유도 CTA
- 사용자 입력에 따른 결과 변경

---

## 4. 사용자 기능 목록

### U-1-1. 포트폴리오 현황 조회

**제공 정보**
- 자산 구성(자산군/종목/비중)
- 기준일(as-of date)
- 데이터 출처 요약

**표시 원칙**
- 최신 active 결과만 표시
- 지연/누락 시 "참고용" 배지 표시

---

### U-1-2. 성과 조회

**조회 항목**
- 기간 수익률(1M/3M/6M/YTD)
- 누적 수익률
- 벤치마크 대비 수익률

**제약**
- LIVE 성과만 노출
- SIM/BACK 성과는 사용자 UI 비노출

---

### U-1-3. 성과 해석 정보

**설명 항목**
- 수익률 산식 요약
- 반영 요소(수수료/환율/배당 여부)
- 기준 가격(종가 기준)

**표현 가이드**
- 단정 표현 금지("우수하다", "권장")
- 설명형 표현 사용("~기준으로 계산됨")

---

### U-1-4. 신뢰 설명 패널(Why Panel)

**구성**
- "이 숫자는 어떻게 계산되었나요?"
- 기준일 / 데이터 스냅샷 요약
- 성과 산출 시점

> execution_id / snapshot_id는 내부적으로 연결하되, 사용자에게는 추상화된 설명만 제공

---

## 5. UX/표현 가이드라인

### 5.1 필수 문구
- "본 정보는 투자 권유가 아닙니다"
- "과거 성과는 미래 수익을 보장하지 않습니다"
- "참고용 정보 제공 목적"

### 5.2 금지 문구
- "추천"
- "지금 투자하세요"
- "수익이 예상됩니다"

---

## 6. API 연계 기준

### 6.1 사용자 API 특성
- 조회 전용(GET)
- 캐시 가능
- 관리자 API와 네임스페이스 분리

### 6.2 참조 기준
- 결과: active result_version
- 성과: LIVE 타입만
- 데이터 기준일 명시 필수

---

## 7. 오류·지연 처리

- 데이터 미산출: 명확한 안내 문구 표시
- 기준일 불일치: 경고 배지
- 시스템 오류: 사용자 친화적 메시지

---

- [ ] 금지 문구 노출 없음
- [ ] 관리자 기능과 명확히 분리
