# Phase 3-C / 사용자 기능 U-2 범위 정의 및 티켓
최초작성일자: 2026-01-18
최종수정일자: 2026-01-18

## 1. 개요
U-2는 U-1의 읽기 전용 사용자 경험을 확장한다. 핵심 원칙은 **Read-only**와 **비권유/비행동 유도** 유지다.

---

## 2. 목표
- 사용자가 성과를 더 쉽게 이해할 수 있도록 조회/비교 기능을 제공한다.
- 결과를 변경하지 않는 범위에서 정보 탐색 편의성을 높인다.

---

## 3. 범위 (초안)

### In-Scope
- 성과 히스토리 조회(기간별 스냅샷 목록)
- 기간 비교 보기(두 기간 성과 비교 요약)
- 지표 상세 보기(산식/반영 요소 상세)
- 사용자 북마크/즐겨찾기(읽기 전용 대상 저장, 결과 변경 없음)

### Out-of-Scope
- 매수/매도/리밸런싱 제안
- 성과 예측/미래 수익 전망
- 사용자 입력 기반 결과 변경
- SIM/BACK 성과 노출

---

## 4. 초기 티켓(안)

### U2-T01 목표/범위 확정
- U-2 In/Out 스코프 확정
- 규제/문구 가드 재확인

### U2-T02 성과 히스토리 조회 API 설계
- 히스토리 조회 기준(일/주/월) 결정
- 응답 스키마 정의

### U2-T03 기간 비교 보기 API 설계
- 비교 대상 기간 선택 규칙
- 비교 요약 필드 정의

### U2-T04 지표 상세 보기 설계
- 지표별 산식/설명 상세 노출 규칙
- 금지 문구 필터 적용

### U2-T05 UI/문구 가이드 업데이트
- U-2 화면별 필수/금지 문구 반영
- 안내 톤 검수

### U2-T06 테스트 및 검증
- Read-only 보장 테스트
- 금지 문구 차단 테스트
- LIVE 성과만 노출 테스트

---

## 5. 의존/전제
- Phase A 입력 스키마 유지
- C-3 성과 지표 정합성 유지
- C-4 권한/가드 정책 유지
