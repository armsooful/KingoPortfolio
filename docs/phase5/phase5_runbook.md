# Phase 5 운영 Runbook v1

## 목적
외부 노출 단계에서 발생할 수 있는 장애에 대한 대응 절차를 표준화한다.

## 공통 대응 절차
1) 영향 범위 파악  
2) 원인 가설 수립  
3) 임시 조치  
4) 원인 확정/조치  
5) 복구 확인  
6) 사후 보고

## 시나리오

### 1. API 오류율 급증
- 증상: 5xx 오류율 상승
- 확인: 최근 배포/로그/대시보드
- 조치: 롤백 또는 트래픽 차단

### 2. 응답 지연
- 증상: p95/p99 급등
- 확인: DB/외부 의존성 상태
- 조치: 스케일 업, 캐시, 재시도 제한

### 3. Guard 차단 실패
- 증상: 추천/선정 경로 호출 성공 로그
- 확인: 규칙/필터/라우팅
- 조치: 즉시 차단 및 배포 중단

## 사후 보고
- 타임라인
- 원인 분석
- 재발 방지
