# Phase 4 장애 대응 Runbook

## 목적
장애 유형별 대응 절차를 표준화한다.

## 공통 대응 절차
1) 상황 인지 및 범위 확인  
2) 원인 가설 수립  
3) 영향 최소화 조치  
4) 원인 확정 및 조치  
5) 복구 확인  
6) 사후 보고

## 시나리오 1: API 응답 지연
- 증상: p95/p99 급등, 타임아웃 증가
- 확인: 메트릭 대시보드, 최근 배포 내역
- 조치: 스케일 업, 캐시 사용 확대, 롤백 검토

## 시나리오 2: 5xx 오류 급증
- 증상: 오류율 급등, 알림 발생
- 확인: 에러 로그/스택트레이스
- 조치: 원인 모듈 격리, 롤백

## 시나리오 3: Guard 경로 실패
- 증상: 차단 실패 로그 발생
- 확인: Guard 룰/필터 검증
- 조치: 즉시 배포 중단 및 룰 수정

## 완료 기준
- 대표 시나리오 3종 포함
