# Phase 3-C / 사용자 기능(U-2) 기능 명세 최종본
최초작성일자: 2026-01-18
최종수정일자: 2026-01-18

## 1. 목적
U-2는 U-1의 읽기 전용 사용자 경험을 확장하여, 성과 이해와 비교를 돕는 추가 조회 기능을 제공한다.
모든 기능은 **Read-only**이며, 행동 유도와 결과 변경을 금지한다.

---

## 2. 범위

### In-Scope
- 성과 히스토리 조회(기간별 스냅샷)
- 기간 비교 보기(두 기간 성과 비교 요약)
- 지표 상세 보기(산식/반영 요소 상세)
- 사용자 북마크/즐겨찾기(읽기 전용 대상 저장)

### Out-of-Scope
- 매수/매도/리밸런싱 제안
- 성과 예측/미래 수익 전망
- 사용자 입력 기반 결과 변경
- SIM/BACK 성과 노출

---

## 3. 사용자 시나리오
- 사용자는 특정 기간의 성과 히스토리를 조회한다.
- 사용자는 두 기간의 성과를 비교한다.
- 사용자는 지표별 산식과 반영 요소를 상세히 확인한다.
- 사용자는 관심 포트폴리오를 북마크한다.

---

## 4. 기능 정의 (초기 설계 기준)

### U-2-1 성과 히스토리 조회
- 기간 단위: 일/주/월 중 선택(기본: 월)
- 반환 필드: 기간, 수익률, 누적 수익률, 기준일, is_reference

### U-2-2 기간 비교 보기
- 비교 대상: 사용자 선택 2개 기간
- 반환 필드: 각 기간 수익률, 차이, 요약 문구(설명형)

### U-2-3 지표 상세 보기
- 지표별 산식/정의/반영 요소 상세 노출
- 금지 문구 필터 및 설명형 톤 유지

### U-2-4 북마크/즐겨찾기
- 사용자별 조회 전용 목록 저장
- 결과값 변경 없음

---

## 5. 표현/규제 가이드
- U-1 문구 가이드 준수
- 필수 문구 및 금지 문구 동일 적용
- 추천/행동 유도/예측 표현 금지

---

## 6. 데이터/보안 기준
- LIVE 성과만 노출
- active result_version 연계
- 관리자 API와 네임스페이스 분리
- 사용자 입력으로 결과 변경 불가

---

## 7. 오류/지연 처리
- 데이터 미산출 시 안내 문구 제공
- 기준일 지연 시 경고 배지 노출
- 시스템 오류는 사용자 친화 메시지로 일괄 처리

---

## 8. 완료 기준
- Read-only 보장
- LIVE 성과만 노출
- 금지 문구 미노출
- 관리자 기능과 분리
