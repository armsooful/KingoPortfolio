## Phase 5 민감 파라미터 마스킹 정책

목적: 로그/응답에 민감 정보가 노출되지 않도록 일관된 마스킹 기준을 제공한다.

### 1) 마스킹 대상
- 인증/보안: 토큰, 세션, API Key, 비밀번호, 인증 코드
- 개인정보: 이메일, 전화번호, 생년월일, 주민번호/식별자
- 금융/계정: 계좌번호, 카드번호, 계정 ID
- 내부 식별자: 내부 키/시드/서명 문자열

### 2) 마스킹 규칙(기본)
- 토큰/키류: 앞 4~6자 + `***` + 뒤 2~4자만 유지
- 이메일: 로컬 파트 일부 + 도메인 유지 (예: `ki***@example.com`)
- 전화번호: 가운데 4자리 `****`
- 계좌/카드: 앞 6자리 + `****` + 뒤 4자리

### 3) 적용 위치
- 응답 페이로드: 민감 필드 제거 또는 마스킹
- 로그: request/response body 기록 시 마스킹 우선
- 예외 로그: 스택트레이스 내 민감 값 제거

### 4) 운영 기준
- 기본 정책은 전 환경 동일
- 예외 발생 시 운영자가 마스킹 기준을 우선으로 판단
- 신규 필드 추가 시 마스킹 여부를 설계/리뷰 단계에서 명시

### 5) 검증 방식(권장)
- 샘플 요청/응답 로그에서 민감 값 노출 여부 점검
- 실제 운영 로그는 원문 저장 금지 (필요 시 보관 정책 분리)
