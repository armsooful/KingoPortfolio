## 1\. 운영 시나리오 기반 E2E 테스트

### 1.1 핵심 시나리오

- 신규 사용자 → 포트폴리오 입력 → 과거데이터 평가 → 결과 확인
- 동일 사용자 → **본인 포트폴리오 간 비교**
- 입력 오류/누락/극단값 시 처리
- 대량 요청/반복 요청(레이트 리밋)

### 1.2 검증 포인트

- 결과 **재현성**(동일 입력 → 동일 결과)
- 타임아웃/재시도 정책 정상 동작
- 로그·감사 이벤트 누락 여부

* * *

## 2\. 결과 표출 문구·UI 안전성 점검

### 2.1 문구 원칙(최종 고정)

- 금지: 추천·권유·성과 보장 암시
- 허용: 과거 데이터 기반 **정보 제공·비교**
- 모든 결과 화면 하단 고정 문구:
    - _“본 결과는 과거 데이터를 기반으로 한 정보 제공이며, 투자 권유 또는 보장을 의미하지 않습니다.”_

### 2.2 UI 구조 점검

- 점수/등급 → **상대적 지표임을 명시**
- 비교 결과 → “우열” 대신 “차이” 중심 표현
- 색상/아이콘으로 투자 판단 유도 금지

* * *

## 3\. 로그·감사·재처리 최종 검증

### 3.1 필수 로그

- 입력 스냅샷 해시
- 평가 버전
- 결과 요약
- 사용자 액션 이벤트

### 3.2 재처리(Replay)

- 동일 입력 재처리 시 결과 불변성 확인
- 버전 변경 시 결과 분리 저장

## 4\. 릴리스 체크리스트

- 모니터링/알림 활성화
