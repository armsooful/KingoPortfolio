# Phase 8-B 데이터 소스 매핑 (v3)

작성일: 2026-01-21  
범위: Phase 8-B (Input Diversity Expansion)

---

## 1. 목적
확장 입력별 데이터 출처와 가능 여부를 명확히 한다.  
데이터 차이는 고지로만 처리하며, 판단 문구는 추가하지 않는다.

---

## 2. 자산군별 가격 데이터
| 자산군 | 기본 데이터 소스 | 비고 |
| --- | --- | --- |
| EQUITY | KRX 일간 시계열 | 국내 주식 |
| BOND | 내부 채권 지표(가용 시) | 미가용 시 중립 오류 |
| COMMODITY | 외부 지표(예: 원자재 인덱스) | 소스 고정 필요 |
| GOLD | 외부 금 가격 지표 | 소스 고정 필요 |
| REIT | 리츠 지수/ETF 프록시 | 소스 고정 필요 |
| ETF | ETF 종가 시계열 | 해외 ETF 포함 |

---

## 3. 배당 포함(총수익) 가능 여부
| 자산군 | TOTAL_RETURN 지원 | 비고 |
| --- | --- | --- |
| EQUITY | 부분 지원(데이터 유무 기준) | 배당 데이터 없으면 PRICE로 처리 |
| BOND | 제한적 지원 | 지표 기반 |
| COMMODITY | 미지원 | PRICE만 제공 |
| GOLD | 미지원 | PRICE만 제공 |
| REIT | 제한적 지원 | 배당 데이터 유무 기준 |
| ETF | 부분 지원 | 배당 재투자 지표 필요 |

---

## 4. 통화 기준 처리
| 통화 | 환산 기준 | 비고 |
| --- | --- | --- |
| KRW | 기본값 | 환산 없음 |
| USD | 환율 시계열 적용 | KRW → USD 변환 |

---

## 5. 누락/오류 처리
- 데이터 누락 시 중립 오류 반환
- “대체 선택 권유” 문구 금지

---

## 6. 고지 문구 예시
- “본 결과는 USD 기준으로 계산되었습니다.”
- “배당을 포함한 총수익 기준입니다.”
