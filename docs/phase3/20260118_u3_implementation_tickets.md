# Phase 3-C / 사용자 기능(U-3) 구현 티켓 및 일정
최초작성일자: 2026-01-18
최종수정일자: 2026-01-18

## 1. 목표
- 고급 사용자 흐름 완성: 반복 사용 시 학습/탐색 효율 개선
- 개인화(비추천/비선정 원칙 유지): 사용자 설정/맥락 반영하되 규제 가드 준수
- 성과 계측 가능화: 행동 -> 지표 -> 피드백의 폐루프 형성

## 2. 범위 요약
- 사용자 설정 고도화: 보기/정렬/필터 프리셋, 알림/노출 빈도 제어(옵트인)
- 상태 인식 UX: 진행 상태/보류 사유/차단 사유 명시, 재시도/대안 경로
- 이력/비교 도구: 사용자 활동 히스토리, 이전 상태 대비 변화 요약
- 가드 강화: 추천/선정 호출 경로 차단 재검증, 금지 키워드 회귀 테스트 자동화
- 비기능: 지연/오류 예산, 접근성 최소 기준, 감사 로그 보존 정책

## 3. 작업 티켓

**전체 상태**: 완료

### U3-01 요구사항 확정 & UX 와이어
- 산출물: 상세 요구사항 정의서, UX 와이어(핵심 플로우)
- 범위: 설정/상태/이력/가드 강화 플로우 전반
- 수용 기준:
  - 기능/비기능 요구사항 확정
  - 비추천/비선정 원칙 반영 확인
  - 주요 화면 플로우 1차 합의
 - 상태: 완료 (UX 와이어 초안 반영)

### U3-02 설정/프리셋 API
- 산출물: API 설계, 스키마 정의, 구현 계획
- 범위: 보기/정렬/필터 프리셋, 알림/노출 빈도 설정
- 수용 기준:
  - 설정 CRUD API 정의
  - 프리셋 기본값/권한 규칙 명시
  - 사용자별 저장/조회 정책 확정
 - 상태: 완료 (API/테스트 반영)

### U3-03 상태/이력 스키마 확장
- 산출물: 상태/이력 이벤트 스키마, 저장 정책
- 범위: 진행 상태, 보류/차단 사유, 사용자 활동 히스토리
- 수용 기준:
  - 상태 전이 정의 및 사유 코드 체계 확정
  - 활동 히스토리 구조 확정
  - 감사 로그 연계 포인트 명시
 - 상태: 완료 (모델/API/테스트 반영)

### U3-04 가드 회귀 테스트 자동화
- 산출물: 테스트 케이스, 자동화 스크립트/테스트
- 범위: 추천/선정 호출 차단, 금지 키워드 회귀
- 수용 기준:
 - 모의 호출 포함 가드 차단 100% 검증
 - 금지 키워드 회귀 테스트 CI 포함
 - 상태: 완료 (금지어 검사 회귀 테스트 추가)

### U3-05 메트릭/알림 연계
- 산출물: 이벤트/메트릭 정의서, 알림 임계치 초안
- 범위: 이탈/재시도/실패율, SLA/오류율/지연 기준
- 수용 기준:
  - 이벤트/메트릭 산식 확정
  - 임계치 초안 및 운영 대응 플로우 정의
 - 상태: 완료 (메트릭/알림 정의서 반영)

### U3-06 문서/릴리즈 노트
- 산출물: U-3 명세/설계 문서, 릴리즈 노트 초안
- 범위: U-3 범위/결정사항 정리, 변경 요약
- 수용 기준:
  - 문서 인덱스 반영
  - 릴리즈 노트 템플릿 확정
 - 상태: 완료 (문서/릴리즈 노트 반영)
