# Phase 4 메트릭/알림 룰 정의

## 목적
핵심 지표를 표준화하고, 장애를 조기에 감지할 수 있는 알림 기준을 확정한다.

## 핵심 메트릭

### 성능
- API p95: < 500ms
- API p99: < 1500ms
- Guard p95: < 200ms

### 안정성
- 5xx 오류율: < 0.1%
- 타임아웃 비율: < 0.5%

### 품질
- 요청 성공률: > 99.9%
- 유효하지 않은 입력 비율: 모니터링만 (경보 제외)

## 알림 룰 (권장)

### Critical
- 5xx 오류율 > 1% (5분 지속)
- API p99 > 2s (5분 지속)
- Guard 경로 5xx 발생 (즉시)

### Warning
- API p95 > 800ms (10분 지속)
- 타임아웃 비율 > 1% (10분 지속)

### Info
- 사용자 오류 급증 (400/422 비율 > 5%)

## 검증 기준
- 알림 룰 적용 후 모의 장애에서 알림 발생 확인
