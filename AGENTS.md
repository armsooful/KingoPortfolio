# AGENTS.md

이 문서는 `/Users/changrim/KingoPortfolio` 프로젝트에서 에이전트가 따라야 할 기본 작업 규칙을 정의합니다.

## 기본 원칙
- 목표를 먼저 명확히 하고, 불확실하면 사용자에게 질문한다.
- 작은 변경으로 빠르게 검증하고, 과도한 수정은 피한다.
- 기존 코드 스타일과 구조를 존중한다.
- 명령 실행 결과는 사용자에게 요약해서 공유한다.

## 작업 흐름
- 변경 전 관련 파일을 확인한다.
- 필요한 경우 `rg`로 빠르게 탐색한다.
- 단일 파일 수정은 `apply_patch`를 우선 사용한다.
- 자동 생성 파일이나 포맷터 출력은 `apply_patch`를 쓰지 않는다.

## 테스트
- 변경 범위에 맞는 최소 테스트를 선택한다.
- 테스트를 실행하지 못한 경우 이유를 명시한다.

## 파일/경로 규칙
- 경로는 항상 인라인 코드로 표기한다. 예: `backend/app/main.py`
- 같은 파일을 여러 번 언급해도 매번 경로를 표기한다.

## 커뮤니케이션
- 상태/결과/다음 단계 순으로 간결하게 보고한다.
- 불확실한 가정은 명시한다.
- 위험한 작업(삭제, 대규모 변경)은 사전 확인한다.
