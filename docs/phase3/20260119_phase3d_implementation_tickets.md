# Phase 3-D / 성과·피드백 루프 구현 티켓
최초작성일자: 2026-01-19
최종수정일자: 2026-01-19

## 1. 목표
- 사용자 행동 → 결과 → 피드백의 폐루프 형성
- 규제 가드 준수(추천/선정/예측 금지)
- 내부 운영 지표 가시화

## 2. 작업 티켓

### D-01 이벤트 스키마 정의
- 산출물: 이벤트 로그 스키마/DDL
- 수용 기준:
  - 이벤트 필드(누가/언제/무엇) 정의
  - 금액/성과 관련 데이터 제외

### D-02 이벤트 수집 API
- 산출물: 이벤트 수집 API 설계/구현
- 수용 기준:
  - 유효성 검증 포함
  - 금지 필드 차단

### D-03 집계 로직(배치/스트림)
- 산출물: 집계 계산 로직/주기 정의
- 수용 기준:
  - 완료율/재시도율/평균 소요 시간 계산
  - 동일 입력 재현 가능

### D-04 피드백 문구 템플릿
- 산출물: 안전 문구 템플릿 세트
- 수용 기준:
  - 비교/평가/예측 금지
  - 사실 서술형만 허용

### D-05 가드·금지 문구 검증
- 산출물: 금지 문구 회귀 테스트
- 수용 기준:
  - 규제 위반 0건
  - CI 포함

### D-06 내부 대시보드(운영용)
- 산출물: 운영 지표 조회 화면/리포트
- 수용 기준:
  - 지표 열람만 가능(조치 권고 금지)

### D-07 문서/운영 가이드
- 산출물: 설계/운영 문서, 릴리즈 노트
- 수용 기준:
  - 문서 인덱스 반영

## 3. 완료 판정 기준
- 이벤트 로그 누락 없음
- 지표 재현 가능(동일 입력 → 동일 결과)
- 피드백 문구 규제 위반 0건
- 추천·선정 경로 0
