## 1. 프로젝트 개요

본 프로젝트는 **투자 추천이나 자문을 제공하지 않고**,  
사용자가 자신의 포트폴리오 결과를 **이해하고 해석할 수 있도록 돕는 설명형 서비스**를 목표로 한다.

핵심 방향:
- 판단 ❌ / 추천 ❌
- 설명 ⭕ / 맥락 제공 ⭕
- 규제 안정성 ⭕

## 3. 단계별 상세 진행 내역

---

### Phase 0. 방향성·규제 정렬

**목표**
- 서비스 성격 명확화
- 투자자문·추천 리스크 사전 차단

**주요 산출물**
- 정보 제공형 서비스 포지션 정의
- 추천/선정/성향 설문 전면 배제
- 전체 프로젝트 규제 가이드 확정

**상태**: 완료

---

### Phase 1. 데이터·기본 시뮬레이션

**목표**
- 포트폴리오 시뮬레이션의 기술적 기반 확보

**주요 산출물**
- 종목/가격/포트폴리오 DB 구조
- 기본 수익률 계산 로직
- 배치 기반 데이터 적재 구조

**상태**: 완료

---

### Phase 2. 핵심 엔진 고도화

**구성 Epic**
- Epic B: 리밸런싱 엔진
- Epic C: 사용자 정의 포트폴리오
- Epic D: 성과 분석

**주요 산출물**
- Rule 기반 리밸런싱 로직
- CAGR, Volatility, MDD 등 성과 지표
- 자동 추천/판단 로직 완전 배제

**상태**: 완료

---

### Phase 3-A. 설명·안심 기능 정의

**목표**
- 숫자를 언어로 번역하는 구조 확립

**주요 산출물**
- 설명 문구 템플릿 라이브러리
- 설명 흐름 중심 UI 와이어프레임
- Phase 3-A API 명세

**상태**: 완료

---

### Phase 3-B. 수익화 검증

**목표**
- 추천 없이도 수익이 가능한지 검증

**주요 산출물**
- 프리미엄 리포트 구조 정의
- 가격·상품 구조 정의서
- 프리미엄 리포트 샘플 1건 완성

**상태**: 완료

---

### Phase A. 설명 자동화 구현 준비

**목표**
- 설명을 시스템적으로 반복 생성 가능하게 함

**주요 산출물**
- Rule/Score 기반 설명 자동 생성 로직 설계
- A 단계 개발 티켓 세트
- Phase A 적용 체크리스트 & 구현 가이드

**상태**: 완료

---

### Phase 3-C. 실데이터 연동 (예정)

**목표**
- 설명 시스템에 신뢰 가능한 실데이터 공급

**구성**
- C-1: 일봉가격/일간수익률 적재
- C-2: DB 기반 시뮬레이션
- C-3: pykrx / Alpha Vantage 연동

**진행 전략**
- Phase A 선적용 후 Phase 3-C 적용
- 설명 로직은 고정, 데이터만 정규화

**상태**: 준비 완료

이제 실데이터를 연결하는 단계 직전”**

## 6. 결론

본 프로젝트는
- 규제 리스크를 통제하면서
- 일반 사용자에게 실제 가치를 제공하고
- 장기적으로 확장 가능한 구조를 갖춘 상태다.

현재까지의 진행은 **순서·내용·깊이 모두 적절**하며,  
다음 단계는 **구현과 데이터 품질 관리**에 집중하면 된다.
