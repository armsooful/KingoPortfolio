# Phase 7 목표

## 1\. Phase 7의 한 문장 목표

> **사용자가 직접 구성한 포트폴리오를 대상으로,  
> 과거 데이터 기반의 성과·리스크를 계산·표출·비교하는  
> 사용자 기능을 최초로 안전하게 공개한다.**

* * *

## 2\. Phase 7의 핵심 목표 (3가지)

### ① 사용자 주도 평가 기능 공개

- 사용자가 **종목/섹터·비중·기간을 직접 선택**
- 시스템은 **계산만 수행**
- 어떠한 추천·판단도 하지 않음

👉 **“선택의 주체는 사용자”를 실제 UX로 구현**

* * *

### ② 과거 데이터 기반 평가 결과의 명확한 표출

- 수익률·변동성·MDD 등 **객관 지표만 제시**
- 모든 결과는 **과거 데이터 기준임을 명시**
- 상태/점수/우열 표현 완전 배제

👉 **평가 결과를 ‘정보’로만 소비 가능하게 만듦**

* * *

### ③ 자기 포트폴리오 간 안전한 비교 제공

- 동일 사용자가 만든 포트폴리오 간 비교만 허용
- 수치·차트 병렬 제시
- 자동 판정·승자·랭킹 금지

👉 **비교는 제공하되, 결론은 사용자가 내리도록 유지**

* * *

## 3\. Phase 7에서 “하지 않는 것” (목표 경계)

Phase 7의 목표는 **무엇을 하지 않는지까지 포함**합니다.

- 종목/비중 추천 ❌
- 최적화/AI 제안 ❌
- 시장/지수 대비 우열 ❌
- 미래 성과 예측 ❌
- 자동 결론 도출 ❌

> **Phase 7은 ‘판단 기능’이 아닌 ‘검증 기능’의 공개 단계**

* * *

## 4\. Phase 7의 성공 기준 (Exit Criteria 관점)

Phase 7은 다음이 충족되면 성공으로 간주한다.

1. 사용자가 스스로 포트폴리오를 만들고 분석 결과를 확인할 수 있다.
2. 모든 결과 화면이 Phase 6의 표현·비교·고지 규칙을 준수한다.
3. 자기 포트폴리오 간 비교가 가능하되, 우열 판단은 제공되지 않는다.
4. 법적·표현적 리스크 없이 외부 사용자에게 공개 가능하다.

* * *

## 5\. Phase 6과의 관계에서 본 Phase 7의 목표

- Phase 6: **경계·규칙을 고정**
- Phase 7: **그 경계 안에서 실제 사용자 기능을 구현**

> **Phase 7은 규칙을 실험하는 단계가 아니라,  
> 규칙을 ‘현실의 UX’로 증명하는 단계다.**

* * *

## 6\. 내부 공유용 한 줄 요약

> **Phase 7의 목표는 “추천 없이도 쓸 수 있는  
> 과거 데이터 기반 포트폴리오 평가 기능을 사용자에게 처음으로 제공하는 것”이다.**
