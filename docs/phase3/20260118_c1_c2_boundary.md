# 운영 장애 vs 데이터 품질 경계 정리
최초작성일자: 2026-01-18
최종수정일자: 2026-01-18

## 1. 정의

### 운영 장애 (Ops Incident)
- 시스템/인프라/배치 실행 자체의 실패
- 예: 실행 실패, 타임아웃, 리소스 부족, 외부 API 장애
- 주 처리: C-1 운영 안정성 영역

### 데이터 품질 (Data Quality)
- 데이터 자체의 정합성/완전성/일관성 문제
- 예: 필수값 누락, 범위 초과, 중복/참조 무결성 위반
- 주 처리: C-2 데이터 품질 영역

## 2. 경계 기준

| 구분 | 운영 장애 | 데이터 품질 |
|---|---|---|
| 원인 | 실행 환경/프로세스 | 데이터 내용/규칙 위반 |
| 감지 | 실행 실패/지연 | Validation FAIL/WARN |
| 조치 | 재시도/장애 복구 | 데이터 보정/재처리 |
| 로그 | batch_execution/ops_alert | validation_result/dq_report |

## 3. 연계 규칙

- Validation FAIL 발생 시
  - C-2에서 데이터 품질 이슈로 기록
  - C-1 실행 상태 FAILED 처리 및 알림 발송

- 운영 장애 발생 시
  - C-1 장애로 기록
  - 원천 데이터 재수집/재처리 필요 시 C-2 흐름으로 전파

## 4. 예시

### 운영 장애 예시
- 배치 작업이 외부 API 타임아웃으로 실패
- 데이터 품질 이슈는 발생하지 않았음

### 데이터 품질 예시
- 가격 데이터 NULL 다수 발생 (REQ FAIL)
- 실행은 정상 완료되었으나 결과는 차단 필요

## 5. 운영 정책

- 장애/품질 분류를 분리하여 리포트
- 동일 실행에서 두 유형이 동시에 발생 가능
