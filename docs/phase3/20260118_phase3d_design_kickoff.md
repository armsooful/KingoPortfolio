# Phase 3-D / 성과·피드백 루프 설계 착수
최초작성일자: 2026-01-18
최종수정일자: 2026-01-18

## 1. 개요
Phase 3-D는 사용자 피드백과 성과 품질을 연결하는 루프를 설계한다. 핵심 원칙은 **설명 품질 개선**과 **운영 가능성 유지**다.

---

## 2. 목표
- 사용자 피드백을 수집·분류·반영하는 경로 정의
- 성과/설명 품질 지표를 측정·개선하는 루프 설계
- 운영자가 통제 가능한 승인/반영 프로세스 마련

---

## 3. 범위 (초안)

### In-Scope
- 사용자 피드백 이벤트 정의(좋아요/도움됨/신고/오류 제보)
- 피드백 수집 API 및 저장 스키마 설계
- 설명 품질 지표 정의(이해도/명확성/중복 여부)
- 개선 반영 프로세스(분류 → 검토 → 승인 → 반영)

### Out-of-Scope
- 자동 학습/자동 수정 반영
- 실시간 개인화 추천
- 성과 예측 모델 도입

---

## 4. 초기 설계 항목(안)

### D1. 피드백 수집 설계
- 이벤트 타입/버전 규칙
- 필수 필드(user_id, portfolio_id, view_name, feedback_type, reason_code)
- 민감 정보 배제 규칙

### D2. 피드백 저장 스키마
- 테이블 구조/인덱스
- 상태 필드(status: NEW/REVIEW/APPROVED/REJECTED)
- 감사 로그 연계

### D3. 품질 지표 정의
- 설명 길이/중복/금지 문구 위반율
- 사용자 반응(도움됨 비율, 신고 비율)
- 기준 임계치 및 알림 규칙

### D4. 운영 프로세스
- 분류/검토 담당자 역할
- 승인 플로우 및 변경 기록
- 롤백/재처리 기준

---

## 5. 산출물(예정)
- Phase 3-D 상세 설계 문서
- 피드백 이벤트 스키마 문서
- 품질 지표 정의서
- 운영 프로세스 체크리스트
