# Phase 7 실행 계획서

## 1\. Phase 7 실행 목표 (재확인)

> **사용자가 직접 구성한 포트폴리오를 대상으로  
> 과거 데이터 기반 성과·리스크를 계산·표출·비교하는  
> 사용자 기능을 법적·표현적 리스크 없이 최초 공개한다.**

- Phase 6 규칙 **절대 준수**
- Phase 7에서는 **기능 구현만 수행**
- 판단·추천·표현 변경 ❌

* * *

## 2\. Phase 7 실행 전략 (핵심 원칙)

### 2.1 실행 원칙

1. **입력 → 계산 → 출력 → 비교**의 최소 루프를 먼저 완성
2. UI 미학보다 **표현 안전성 우선**
3. 기능 추가보다 **재현성·일관성 우선**
4. Phase 6 위반 시 → 즉시 중단/수정

* * *

## 3\. 단계별 실행 계획 (Step-by-Step)

* * *

### STEP 1. 입력 파이프라인 구현 (Week 1)

**목적**  
사용자가 “시스템 개입 없이” 포트폴리오를 완성할 수 있도록 한다.

**작업 내용**

- 종목/섹터 선택 UI 구현
- 비중 입력 및 합계 검증
- 분석 기간 / 리밸런싱 가정 입력

**산출물**

- 입력 UI 컴포넌트
- 입력 데이터 스키마 v2
- 중립 오류 메시지 세트

**완료 기준**

- 기본값·추천값 없음
- 합계 100% 미충족 시 진행 불가
- 미래 시제/판단 문구 0건

* * *

### STEP 2. 과거 데이터 평가 엔진 연결 (Week 2)

**목적**  
입력된 구성으로 **결정론적 평가 결과**를 생성한다.

**작업 내용**

- 과거 가격 데이터 로딩
- 포트폴리오 시계열(NAV) 계산
- 지표 계산(CAGR, 변동성, MDD)

**산출물**

- 평가 엔진 모듈
- 계산 로그(재현성 확인용)

**완료 기준**

- 동일 입력 → 동일 출력
- Output Schema v2 100% 준수
- 데이터 누락 시 중립 오류 처리

* * *

### STEP 3. 결과 표출 UI 구현 (Week 3)

**목적**  
계산 결과를 **정보 형태로만** 사용자에게 전달한다.

**작업 내용**

- 결과 요약 화면(표)
- 시계열 차트(NAV, 누적수익, MDD)
- 고지 문구 전 화면 고정 삽입

**산출물**

- 결과 화면 UI
- 차트 컴포넌트

**완료 기준**

- 점수/우열/강조 표현 0건
- ui\_guard\_rules\_v2.md 완전 준수
- 고지 문구 숨김 불가

* * *

### STEP 4. 자기 포트폴리오 간 비교 구현 (Week 4)

**목적**  
“비교는 가능하되, 판단은 제공하지 않는다”를 UX로 증명한다.

**작업 내용**

- 동일 사용자 포트폴리오 선택
- 수치 병렬 비교
- 동일 축 차트 병렬 표시

**산출물**

- 비교 화면
- 비교 대상 검증 로직

**완료 기준**

- 다른 사용자 비교 불가
- 자동 판정/승자/랭킹 없음
- comparison\_rules\_v2.md 준수

* * *

### STEP 5. 저장·히스토리 기능 (Week 4 병행)

**목적**  
재방문과 반복 검증이 가능하도록 한다.

**작업 내용**

- 포트폴리오 저장
- 평가 결과 히스토리 조회

**산출물**

- 저장 API
- 히스토리 화면

**완료 기준**

- 동일 결과 재현
- 저장 시 판정/권유 문구 없음

* * *

### STEP 6. Phase 6 준수 통합 검증 (Week 5)

**목적**  
Phase 7이 Phase 6 경계를 넘지 않았음을 **기계적으로 증명**한다.

**작업 내용**

- 금지어 자동 스캔 연동
- Golden Test CI 통합
- 실행 스크립트: `scripts/phase7_compliance_check.sh`
- Phase 6 검수 리포트 작성

**산출물**

- 테스트 리포트
- Phase 6 검수 리포트(승인본)

**완료 기준**

- Golden Test 100% 통과
- Phase 6 종료 승인 유지

* * *

## 5\. Phase 7 완료 기준 (Exit Criteria)

Phase 7은 아래 조건 충족 시 종료한다.

- 사용자가 포트폴리오를 직접 구성·분석 가능
- 과거 데이터 기반 지표 정상 출력
- 자기 포트폴리오 간 비교 가능
- Phase 6 규칙 위반 0건
- 외부 공개 가능 상태

* * *

> ‘규칙을 지킨 상태로 실제 사용 가능한 기능을 처음 공개하는 단계’다.**
