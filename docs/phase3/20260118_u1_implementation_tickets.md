# Phase 3-C / Epic U-1 구현 티켓
최초작성일자: 2026-01-18
최종수정일자: 2026-01-18

## 개요

Phase 3-C 사용자 기능 Epic U-1(읽기 전용 사용자 기능) 구현을 위한 작업 티켓 목록이다. 모든 기능은 Read-only이며, 사용자 행동 유도를 금지한다.

---

## 티켓 목록

### U1-T01 전제 조건 검증 체크리스트 정리
- C-1~C-4 완료 증빙 확인
- 사용자 노출 승인 기준 명문화
- 검증 결과 기록 포맷 정의

### U1-T02 API 범위/제약 명세
- Read-only(GET) 전용
- active result_version만 노출
- LIVE 성과만 허용
- 기준일(as-of)·출처 필드 필수화

### U1-T03 사용자 포트폴리오 현황 조회 API
- 자산 구성/비중/기준일/출처 응답 정의
- 최신 active 결과만 반환
- 지연/누락 시 참고용 배지 처리 규칙 반영

### U1-T04 사용자 성과 조회 API
- 기간 수익률(1M/3M/6M/YTD)
- 누적 수익률/벤치마크 대비 수익률
- LIVE 타입만 노출

### U1-T05 성과 해석 정보 API
- 수익률 산식 요약
- 반영 요소(수수료/환율/배당) 설명
- 기준 가격(종가 기준) 명시

### U1-T06 Why Panel 데이터/콘텐츠 설계
- 계산 시점/스냅샷 요약 제공
- execution_id/snapshot_id 내부 연결 유지
- 사용자 화면에는 추상화된 설명만 노출

### U1-T07 UX 문구 적용
- 필수 문구 적용
  - "본 정보는 투자 권유가 아닙니다"
  - "과거 성과는 미래 수익을 보장하지 않습니다"
  - "참고용 정보 제공 목적"
- 금지 문구 차단 규칙 정의
  - "추천", "지금 투자하세요", "수익이 예상됩니다"

### U1-T08 오류/지연 처리 메시지 정의
- 데이터 미산출: 명확한 안내 문구
- 기준일 불일치: 경고 배지
- 시스템 오류: 사용자 친화적 메시지

### U1-T09 캐시 정책 정의
- 조회 캐시 TTL 설정
- 무효화 조건 정의(새 active 버전 생성 등)

### U1-T10 관리자 API 분리 검증
- 사용자/관리자 네임스페이스 분리
- 권한 우회 방지 검증

### U1-T11 테스트 케이스 작성
- Read-only 보장 테스트
- LIVE 성과만 노출 테스트
- 금지 문구 노출 차단 테스트
- 오류 메시지 표준화 검증

### U1-T12 문서 산출물 작성
- 사용자 API 명세서
- UI 시안/와이어프레임
- 사용자 기능 검증 체크리스트

---

## 완료 기준

- 모든 사용자 기능이 Read-only로 동작
- LIVE 성과만 노출
- 기준일/출처/산식 설명 제공
- 금지 문구 노출 없음
- 관리자 기능과 명확히 분리
