## Batch/Schedule Monitoring Policy

목적: 배치 실패/지연을 조기 탐지하고 운영 대응 시간을 단축한다.

### 1) 대상 작업
- 일/주간 집계 배치
- 데이터 적재/정합성 검사
- 알림/리포트 생성 작업

### 2) 모니터링 기준
- 성공률: 99.5% 이상
- 실행 지연: 계획 시간 대비 10분 이상 지연 시 경고
- 실패 횟수: 연속 2회 실패 시 장애 알림

### 3) 지표/로그
- 배치 시작/완료 시각
- 처리 건수
- 실패 원인(에러 코드/메시지)
- 재시도 횟수

### 4) 알림 연동
- 실패/지연 발생 시 `ops_alerting.yaml` 기준에 따라 알림
- 배치 재시도 1회 실패 시 즉시 경고, 2회 실패 시 장애

### 5) 운영 대응
- 최근 성공 배치 기준 롤백 가능 여부 확인
- 데이터 정합성 검증 후 재실행
- 동일 원인 반복 시 배치 중지 및 핫픽스 전환
