# Phase 4 롤백 시나리오 및 절차

## 목적
장애 발생 시 빠르게 안정 상태로 복구하기 위한 표준 절차를 제공한다.

## 롤백 트리거
- 배포 직후 5xx 오류율 급증
- SLA p99 기준 2배 이상 초과
- 핵심 플로우 장애(로그인/조회/진단)

## 롤백 절차
1) 배포 중단 및 신규 배포 차단  
2) 이전 안정 버전으로 롤백  
3) 데이터 마이그레이션 영향 확인  
4) 핵심 플로우 스모크 테스트  
5) 운영 공지 및 사후 보고

## 검증 체크리스트
- `/health` 정상 응답
- 주요 API 응답 정상
- Guard 경로 차단 정상
- 로그/알림 정상 수집

## 완료 기준
- 롤백 절차 문서화 완료
- 모의 롤백 시나리오 1회 이상 검증
