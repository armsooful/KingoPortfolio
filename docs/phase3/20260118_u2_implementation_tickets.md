# Phase 3-C / Epic U-2 구현 티켓
최초작성일자: 2026-01-18
최종수정일자: 2026-01-18

## 개요
U-2는 U-1의 읽기 전용 사용자 기능을 확장한다. 모든 기능은 Read-only이며, 금지 문구/행동 유도 표현을 금지한다.

---

## 티켓 목록

### U2-T01 요구사항 확정
- U-2 기능 명세 최종본 검토/승인
- In/Out 스코프 고정

### U2-T02 성과 히스토리 조회 API 설계/구현
- 기간 단위(일/주/월) 규칙 확정
- 응답 스키마 정의 및 구현
- LIVE 성과만 노출
 - 설계 문서: docs/phase3/20260118_u2_performance_history_api_design.md

### U2-T03 기간 비교 보기 API 설계/구현
- 비교 기간 입력 규칙 정의
- 요약 문구 템플릿 적용(설명형)
- 금지 문구 필터 적용
 - 설계 문서: docs/phase3/20260118_u2_period_comparison_api_design.md

### U2-T04 지표 상세 보기 설계/구현
- 지표별 산식/정의/반영 요소 노출
- 설명형 톤 유지, 금지 문구 필터 적용
 - 설계 문서: docs/phase3/20260118_u2_metric_detail_api_design.md

### U2-T05 북마크/즐겨찾기 저장
- 사용자별 저장/조회(읽기 전용 대상)
- 결과 변경 없음 보장
- 설계 문서: docs/phase3/20260118_u2_bookmark_design.md
 - DDL 문서: docs/phase3/20260118_u2_bookmark_ddl.md

### U2-T06 UX 문구 가이드 적용
- 필수 문구/금지 문구 적용 확인
- 사용자 화면 표현 검수

### U2-T07 테스트/검증
- Read-only 보장 테스트
- LIVE 성과만 노출 테스트
- 금지 문구 차단 테스트
- 오류/지연 처리 메시지 검증

---

## 완료 기준
- U-2 모든 기능이 Read-only로 동작
- LIVE 성과만 노출
- 금지 문구 미노출
- U-1 문구 가이드 준수
